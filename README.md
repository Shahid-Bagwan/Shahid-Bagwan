## How I think

<table>
<tr>
<td width="60%" valign="top">

#### First principles, problem-first

Diagnosis before solution. Most "performance issues" turn out to be visibility issues; most "feature requests" turn out to be a missing constraint. The fix is worthless if it's aimed at the wrong bottleneck.

#### Agentic system design

Multi-step pipelines with explicit lanes, durable state, layered guards, and human-in-the-loop where it matters. *Account safety beats growth, every time.*

#### Built for fast-learning startup environments

Small teams, ambiguous problems, real production constraints. The work that lands is rarely the work that was specified — so the system has to be cheap to redirect, not expensive to defend.
